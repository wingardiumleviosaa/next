[MySQL] 在 Ubuntu18.04 建立 MySQL NDB Cluster 8.0

Publish on: 2020-08-11 Classify at: DataEngineering/Database
Words: 2068 Read:≈ 5mins

MySQL NDB Cluster

MySQL cluster 將標準的 MySQL server 與內存中叢集式儲存引擎 NDB 集成起來，允許在多個無共享的系統中部署「內存中」資料庫的叢集。

叢集通常會有多台機器，每台運行不同程序，包括 MySQL Server、NDB 叢集的數據節點、管理伺服器，以及可能專門的數據訪問程式，如下圖：

架構中的三個角色

  • 管理節點 Management Node
    用於管理叢集內的其他節點，如提供配置數據、啟動並停止節點、運行備份等。由於這類節點負責管理其他節點的配置，應在啟動其他節點之前首先啟動這類節點。

  • 數據節點 Data Node
    用於保存叢集的數據，資料寫在 RAM 或 Disk。當節點有 2 個以上時就能實現集群的高可用保證，不過節點增加，集群的處理速度會變慢。

  • SQL 節點 (API)
    用來訪問叢集數據，負責 SQL 的 Table schema，提供對外應用服務。增加 API 節點會提高整個集群的並發訪問速度和整體的吞吐量，該節點可以部署在 Web 應用伺服器上，也可以部署在專用的伺服器上。

環境準備

搭建 MySQL Cluster 至少要一個管理節點來管理，一個 SQL 節點來實現 MySQL server 功能和兩個資料節點實現 NDB Cluster 的功能。此篇文章將測試使用雙 SQL節點來搭建測試環境：

1a) Management Node  10.1.5.31
2b) Data Node        10.1.5.32
3c) Data Node        10.1.5.33
4d) SQL Node         10.1.5.32
5e) SQL Node         10.1.5.33

以上機器使用 Ubuntu 18.04 作業系統。

Management Node

下載並安裝 

1$ cd ~
2$ wget https://dev.mysql.com/get/Downloads/MySQL-Cluster-8.0/mysql-cluster-community-management-server_8.0.21-1ubuntu18.04_amd64.deb
3$ sudo dpkg -i mysql-cluster-community-management-server_8.0.21-1ubuntu18.04_amd64.deb

編輯配置檔 config.ini 

1$ sudo mkdir /var/lib/mysql-cluster
2$ sudo vi /var/lib/mysql-cluster/config.ini


 1[ndbd default]
 2NoOfReplicas=2
 3
 4[ndb_mgmd]
 5NodeId=1
 6hostname=10.1.5.31
 7datadir=/var/lib/mysql-cluster
 8
 9[ndbd]
10NodeId=2
11hostname=10.1.5.32
12datadir= /usr/local/mysql/data
13
14[ndbd]
15NodeId=3
16hostname=10.1.5.33
17datadir= /usr/local/mysql/data
18
19[mysqld]
20NodeId=4
21hostname=10.1.5.32
22
23[mysqld]
24NodeId=5
25hostname=10.1.5.33

配置說明如下：

 1[ndb default]
 2# data node 的配置
 3NoOfReplicas=2 # 數據節點數
 4
 5[ndb_mgmd]
 6# 管理節點配置
 7hostname=10.1.5.31
 8datadir=/var/lib/mysql-cluster # log files 儲存目錄
 9
10[ndbd] # 數據節點配置
11hostname= # Remote IP address
12NodeId=2 # 為管理各節點而命名的 id，從 2 依序編碼
13datadir=/usr/local/mysql/data  # 存放資料的目錄
14
15[mysqld]
16# SQL node
17hostname= # Remote IP address
18NodeId=3

測試是否能啟動 

1$ sudo ndb_mgmd -f /var/lib/mysql-cluster/config.ini

會出現以下訊息：

1MySQL Cluster Management Server mysql-8.0.21 ndb-8.0.21

代表啟用成功。

補充
如果啟動後需要修改 cluster 內的 IP ，更新完 config.ini 後，啟動時改成下面指令：

1$ ndb_mgmd -f /var/lib/mysql-cluster/config.ini configdir=/var/lib/mysql-cluster --reload --initial

將程序設為 service

先關掉剛剛測試的程序，

1$ sudo pkill -f ndb_mgmd

建立服務檔，

1$ sudo vi /etc/systemd/system/ndb_mgmd.service

貼上以下內容：

 1[Unit]
 2Description=MySQL NDB Cluster Management Server
 3After=network.target auditd.service
 4
 5[Service]
 6Type=forking
 7ExecStart=/usr/sbin/ndb_mgmd -f /var/lib/mysql-cluster/config.ini --configdir=/var/lib/mysql-cluster/
 8ExecReload=/bin/kill -HUP $MAINPID
 9KillMode=process
10Restart=on-failure
11
12[Install]
13WantedBy=multi-user.target

重新載入設定檔並將服務設為開機啟動，最後啟動服務。

1$ sudo systemctl daemon-reload
2$ sudo systemctl enable ndb_mgmd
3$ sudo systemctl start ndb_mgmd

查案服務狀態，

1$sudo systemctl status ndb_mgmd

如果得到以下回復代表 management node 已經成功啟動。

Data Node

安裝並下載依賴程式 

1$ cd ~
2$ wget https://dev.mysql.com/get/Downloads/Mysql-Cluster-8.0/mysql-cluster-community-data-node_8.0.21-1ubuntu18.04_amd64.deb
3$ sudo apt update
4$ sudo apt install -y libclass-methodmaker-perl
5$ sudo dpkg -i mysql-cluster-community-data-node_8.0.21-1ubuntu18.04_amd64.deb

配置

建立mysql配置檔 my.cnf

1$ vi /etc/my.cnf

貼上以下內容：

1[mysql_cluster]
2ndb-connectstring=10.1.5.31

其中 ndb-connectstrings 代表 cluster manager 的位址。

根據管理者的配置，建立data node的數據存放目錄。

1$ sudo mkdir -p /usr/local/mysql/data

測試是否能啟動 

1$ sudo ndbd

會出現以下訊息，代表啟用成功。

12020-08-05 15:31:59 [ndbd] INFO     -- Angel connected to '10.1.5.31:1186'
22020-08-05 15:31:59 [ndbd] INFO     -- Angel allocated nodeid: 2

先關掉剛剛測試的程序，

1$ sudo pkill -f ndbd

建立服務檔，

1$ sudo vi /etc/systemd/system/ ndbd.service

貼上以下內容：

 1[Unit]
 2Description=MySQL NDB Data Node Daemon
 3After=network.target auditd.service
 4
 5[Service]
 6Type=forking

12[Install]
13WantedBy=multi-user.target

2$ sudo systemctl enable ndbd

SQL Node

下載

因為也是跟 data node 一樣把壓縮檔放在家目錄下，所以另外建一個資料夾 install 存放 SQL Node 的安裝檔。

1$ cd ~
2$ wget https://dev.mysql.com/get/Downloads/Mysql-Cluster-8.0/mysql-cluster_8.0.21-1ubuntu18.04_amd64.deb-bundle.tar
3$ mkdir install
4$ tar -xvf mysql-cluster_8.0.21-1ubuntu18.04_amd64.deb-bundle.tar -C install/
5$ cd install

下載依賴包 

1$ sudo apt update
2$ sudo apt install -y libaio1 libmecab2

開始安裝 MySQL Cluster dependencies 

1$ sudo dpkg -i mysql-common_8.0.21-1ubuntu18.04_amd64.deb
2$ sudo dpkg -i mysql-cluster-community-client-core_8.0.21-1ubuntu18.04_amd64.deb
3$ sudo dpkg -i mysql-cluster-community-client_8.0.21-1ubuntu18.04_amd64.deb
4$ sudo dpkg -i mysql-client_8.0.21-1ubuntu18.04_amd64.deb
5$ sudo dpkg -i mysql-cluster-community-server-core_8.0.21-1ubuntu18.04_amd64.deb
6$ sudo dpkg -i mysql-cluster-community-server_8.0.21-1ubuntu18.04_amd64.deb

在安裝mysql-cluster-community-server 的過程中，會要求輸入 mysql root 的密碼。

之後會詢問是否要使用 mysql 8.0 提供的新功能加強密碼加密，

這邊選擇保持原本的密碼方式， 兼容性較高。（此設置在安裝完畢後可依需求改變）

安裝 MySQL server binary。

1$ sudo dpkg -i mysql-server_8.0.21-1ubuntu18.04_amd64.deb

重配置 my.cnf 檔案 

1$ sudo vi /etc/mysql/my.cnf

在末端加上以下內容：

1[mysqld]
2ndbcluster
3
4[mysql_cluster]
5ndb-connectstring=10.1.5.31
6bind-address= 0.0.0.0

配置檔內容說明如下：

 1[mysqld]
 2# mysqld 程序的配置
 3ndbcluster
 4# 執行 ndb 儲存引擎
 5
 6[mysql_cluster]
 7ndb-connectstring=10.1.5.31
 8# management node 位址
 9bind-address= 0.0.0.0
10# 代表可提供外部網路連線 remote 管理，若沒有此參數的話，SQL 只能在 local 端使用

重啟 mysql service 套用以上變更 

1$ sudo systemctl restart mysql

加入開機啟動。

1$ sudo systemctl enable mysql

使用遠端 workblench 進行連線

先進入 SQL node 在本地端新建一個使用者並賦予存取所有位置的權限：

1$ mysql -u root -p


1mysql > CREATE USER 'user'@'%' IDENTIFIED BY 'password';
2mysql > GRANT ALL ON *.* TO 'user'@'%';

開啟 MySQLworkblench 新增連線對象為 cluster 內任何一個 SQL node，便可成功連線。
