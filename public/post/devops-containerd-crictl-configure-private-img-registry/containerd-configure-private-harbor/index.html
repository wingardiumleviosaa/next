Containerd (crictl) 配置私有鏡像倉庫

Publish on: 2022-03-16 Classify at: DevOps/Docker
Words: 347 Read:≈ 1mins

前幾天將 k8s 升級到 1.23 版，使用 containerd 當 CRI，立馬就遇到要拉取私有 image registry 的狀況，本文紀錄配置過程。

配置 containerd config.toml

依據遠端私有倉庫是否加密，有不同配置：

- 私有倉庫帶有 tls 加密 (https) 

1vim /etc/containerd/config.toml

在 [plugins."io.containerd.grpc.v1.cri".registry.mirror] 下加上私有倉庫位址/URL

1[plugins."io.containerd.grpc.v1.cri".registry.mirrors."harbor.nexmasa.com"]
2   endpoint = ["https://harbor.nexmasa.com"]

另外再接著加上 [plugins."io.containerd.grpc.v1.cri".registry.configs] 的項目

1[plugins."io.containerd.grpc.v1.cri".registry.configs]
2   [plugins."io.containerd.grpc.v1.cri".registry.configs."harbor.nexmasa.com".tls]
3      ca_file   = "/etc/containerd/harbor.nexmasa.com/ca.pem"
4      cert_file = "/etc/containerd/harbor.nexmasa.com/cert.pem"
5      key_file  = "/etc/containerd/harbor.nexmasa.com/key.pem"
6   [plugins."io.containerd.grpc.v1.cri".registry.configs."harbor.nexmasa.com".auth]
7      username = "xxxxxx"
8      password = 'xxxxxx'

- 私有倉庫無加密 (http) 或是想略過憑證檢查

在 tls 配置項下，以 insecure_skip_verify = true 參數取代上面配置的金鑰及憑證。

1[plugins."io.containerd.grpc.v1.cri".registry.configs]
2   [plugins."io.containerd.grpc.v1.cri".registry.configs."harbor.nexmasa.com".tls]
3      insecure_skip_verify = true

重啟 containerd 

1systemctl restart containerd
