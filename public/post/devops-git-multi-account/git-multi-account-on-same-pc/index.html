[Git] 在同一台電腦中設置多個 Git 帳號

Publish on: 2020-08-05 Classify at: DevOps/Git
Words: 454 Read:≈ 1mins

為每個帳號產生 ssh key 

1$ ssh-keygen -t rsa -C "userName@address"

產生完畢後，將公鑰放到對應帳號的 github 中

將新金鑰新增到 SSH agent 中

因為作業系統預設只讀取私鑰 id_rsa，為了讓 SSH 識別新的私鑰，需將其新增到 SSH agent中：

1$ ssh-add ~/.ssh/金鑰名稱

如果出現 Could not open a connection to your authentication agent 的錯誤，就試著用以下命令：

1$ ssh-agent bash
2$ ssh-add ~/.ssh/金鑰名稱

Warning

重啟電腦後都需要重新 ssh-add。因為這個命令不會永久性的記住私鑰。使用 ssh-add 會把指定的私鑰新增到 ssh-agent 所管理的一個 session 當中。而 ssh-agent 是一個用於儲存私鑰的臨時性的 session 服務，也就是說當你重啟之後，ssh-agent 服務便會重置，session 會話也會失效。

解決每次重啟都要 ssh-add 的問題

在 git 安裝目錄下的 etc/bash.bashrc 文件中末加入

1eval "$(ssh-agent -s)"
2ssh-add ~/.ssh/金鑰名稱

配置多個 ssh-key

修改 ~/.ssh/config 文件

1#default github
2Host github.com
3HostName github.com
4IdentityFile ~/.ssh/id_rsa
5
6Host github_second
7HostName github.com
8IdentityFile ~/.ssh/id_rsa_second

其中 Host 為 HostName 的別名。

測試連線 

1$ ssh -T git@[Host]

出現

1Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access.

代表連線成功

在專案下設配置使用者

先取消 global

1$ git config --global --unset user.name
2$ git config --global --unset user.email

設置 repo 自己的 user & email

1$ git config  user.email "xxxx@xx.com"
2$ git config  user.name "xxxx"

clone 遠端資料

1$ git clone git@[Host]:UserName/repositoryName

git push

1$ git remote add origin git@[Host]:UserName/repositoryName.git
2$ git add -am "commit msg"
3$ git push origin master
