使用 docker-compose 建立 Harbor 私有倉庫 (w/ https)

Publish on: 2022-04-10 Classify at: DevOps/Docker
Words: 1070 Read:≈ 3mins

默認情況下，Harbor 不附帶證書，可以直接使用 http 訪問。但在正式上線的環境中，建議配置 https。

準備 https 所需證書

在生產環境中，建議使用由受信任的第三方 CA 簽名的證書。在測試或開發環境中，則可以使用自簽的 CA。以下範例為使用 openssl 生成 CA，分為使用 DNS 或是直接使用 IP。

使用域名(DNS)

  1. 準備 x509 v3 服務檔，假設 harbor 伺服器使用的 DNS 為 harbor.ula.com。
 1cat > v3.ext <<-EOF
 2authorityKeyIdentifier=keyid,issuer
 3basicConstraints=CA:FALSE
 4keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
 5extendedKeyUsage = serverAuth
 6subjectAltName = @alt_names
 7
 8[alt_names]
 9DNS.1=harbor.ula.com
10DNS.2=ula.com
11DNS.3=harbor
12EOF
  1. 準備金鑰及證書的 shell script
1vim prepare_ssl.sh

貼上以下內容

 1#!/bin/bash
 2
 3CURDIR="`pwd`"/"`dirname $0`"
 4HOSTNAME=$1
 5DOMAIN=$2
 6
 7INSTALL_PATH="/data/certs"
 8mkdir -p $INSTALL_PATH
 9
10cd $INSTALL_PATH
11# generate CA certificate private key
12openssl genrsa -out ca.key 4096
13
14# generate CA certificate
15openssl req -x509 -new -nodes -sha512 -days 3650 -subj "/C=TW/ST=Taipei/L=Taipei/O=$HOSTNAME/OU=Personal/CN=$HOSTNAME.$DOMAIN" -key ca.key -out ca.crt
16
17# generate server's private key
18openssl genrsa -out $DOMAIN.key 4096
19
20# Generate a certificate signing request (CSR)
21openssl req -sha512 -new -days 3650 -subj "/C=TW/ST=Taipei/L=Taipei/O=$HOSTNAME/OU=Personal/CN=$HOSTNAME.$DOMAIN" -key $DOMAIN.key -out $DOMAIN.csr
22
23# use v3.ext to generate certificate for harbor host
24openssl x509 -req -sha512 -days 3650 -extfile $CURDIR/v3.ext -CA ca.crt -CAkey ca.key -CAcreateserial -in $DOMAIN.csr -out $DOMAIN.crt
25
26# Convert *.crt to *.cert, for use by Docker.
27openssl x509 -inform PEM -in $DOMAIN.crt -out $DOMAIN.cert
28chmod 400 $DOMAIN.key
  1. 執行腳本創建相關證書及金鑰
1./prepare_ssl.sh harbor ula.com

使用 IP

  1. 準備 x509 v3 服務檔
1cat > v3.ext <<-EOF
2authorityKeyIdentifier=keyid,issuer
3basicConstraints=CA:FALSE
4keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
5extendedKeyUsage = serverAuth
6subjectAltName = IP:10.1.5.142
7EOF
  1. 準備金鑰及證書的 shell script
1vim prepare_ssl.sh

貼上以下內容

 1#!/bin/bash
 2
 3CURDIR="`pwd`"/"`dirname $0`"
 4IP=$1
 5
 6INSTALL_PATH="/data/certs"
 7mkdir -p $INSTALL_PATH
 8
 9cd $INSTALL_PATH
10# generate CA certificate private key
11openssl genrsa -out ca.key 4096
12
13# generate CA certificate
14openssl req -x509 -new -nodes -sha512 -days 3650 -subj "/C=TW/ST=Taipei/L=Taipei/O=$IP/OU=Personal/CN=$IP" -key ca.key -out ca.crt
15
16# generate server's private key
17openssl genrsa -out $IP.key 4096
18
19# Generate a certificate signing request (CSR)
20openssl req -sha512 -new -days 3650 -subj "/C=TW/ST=Taipei/L=Taipei/O=$IP/OU=Personal/CN=$IP" -key $IP.key -out $IP.csr
21
22# use v3.ext to generate certificate for harbor host
23openssl x509 -req -sha512 -days 3650 -extfile $CURDIR/v3.ext -CA ca.crt -CAkey ca.key -CAcreateserial -in $IP.csr -out $IP.crt
24
25# Convert *.crt to *.cert, for use by Docker.
26openssl x509 -inform PEM -in $IP.crt -out $IP.cert
27chmod 400 $IP.key
  1. 執行腳本創建相關證書及金鑰
1./prepare_ssl.sh 10.1.5.142

因為本次建立沒有特別新增域名，所以以下範例是使用 IP 位址建立。

開始安裝

下載安裝檔 

1cd ~
2wget https://github.com/goharbor/harbor/releases/download/v2.4.2/harbor-offline-installer-v2.4.2.tgz
3tar zxvf harbor-offline-installer-v2.4.2.tgz

修改 harbor 配置檔 

1cd ~/harbor
2cp harbor.yml.tmpl harbor.yml
3vim harbor.yml

針對以下三處修改

 1# 沒有域名，寫 harbor 所在伺服器 ip
 2hostname: 10.1.5.142
 3http:
 4# 因主機有其他服務已佔用 80 port 所以改成 8080
 5  port: 8080
 6https:
 7# 因主機有其他服務已佔用 443 port 所以改成 4433
 8  port: 4433
 9  # 設定證書及金鑰路徑
10  certificate: /data/certs/10.1.5.142.crt
11  private_key: /data/certs/10.1.5.142.key
12
13# harbor 預設登入密碼
14harbor_admin_password: Harbor12345

配置 docker 以登入私有倉庫

建立 docker 對應倉庫的證書存放目錄

1mkdir -p /etc/docker/certs.d/10.1.5.142:4433
2cp 10.1.5.142.key /etc/docker/certs.d/10.1.5.142:4433/
3cp 10.1.5.142.cert /etc/docker/certs.d/10.1.5.142:4433/
4cp ca.crt /etc/docker/certs.d/10.1.5.142:4433/

重啟 docker

1systemctl restart docker

登入遠端倉庫

1docker login 10.1.5.142:4433 -u admin -p Harbor12345
2WARNING! Using --password via the CLI is insecure. Use --password-stdin.
3WARNING! Your password will be stored unencrypted in /root/.docker/config.json.
4Configure a credential helper to remove this warning. See
5https://docs.docker.com/engine/reference/commandline/login/#credentials-store
6
7Login Succeeded

1docker pull busybox
2docker tag busybox:latest 10.1.5.142:4433/test/busybox:latest
3docker push 10.1.5.142:4433/test/busybox:latest
