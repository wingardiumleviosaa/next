[Docker] Run Cronjob as Non Root User in Alpine Container

Publish on: 2022-10-23 Classify at: DevOps/Docker
Words: 158 Read:≈ 1mins

TL;DR

apline image 預設只能給 root 執行 crond，但剛好遇到有不允許 container 使用 root 執行的安全政策的情況。本篇記錄如何在 apline container 中使用 non root user 執行 crond。

Dockerfile 

 1FROM harbor.wistron.com/base_image/alpine:3.12
 2
 3USER root
 4RUN apk update && apk --no-cache add dcron libcap
 5
 6RUN addgroup --gid 1000 ula && adduser --disabled-password --ingroup ula --uid "1000" ula
 7
 8RUN chown ula:ula /usr/sbin/crond && setcap cap_setgid=ep /usr/sbin/crond
 9
10COPY --chown=ula:ula init.sh /home/ula/app
11RUN touch /etc/crontabs/ula && chown -R ula:ula /etc/crontabs/ula
12
13USER ula
14
15# set scheduler
16RUN echo "*/10 * * * * date >> /home/ccoebot/app/sync.log" >> /etc/crontabs/ula
17
18# This container exposes port 8080 to the outside world
19EXPOSE 8080
20
21CMD ["/home/ula/app/init.sh"]

init.sh 

1#!/bin/bash
2crond -l 8
