Terraform Connection to Kuberentes Refused Error

Publish on: 2021-08-29 Classify at: DevOps/Terraform
Words: 472 Read:≈ 1mins

問題

在 terrform apply 的時候一直卡在 Error: Post "http://localhost/api/v1/namespaces": dial tcp [::1]:80: connect: connection refused 的錯誤

原因

看起來是沒有正確的抓到 Provider 裡面設的 kube config file

原本的腳本如下

1// main.tf
2provider "helm" {
3  kubernetes {
4    config_path = "/root/.kube/config"
5  }
6}


1// kubernetes.tf
2provider "kubernetes" {}

解法

修改腳本如下

1// main.tf
2provider "kubernetes" {
3    host = "https://10.1.5.140:8443"
4
5    client_certificate     = file("~/.kube/client.pem")
6    client_key             = file("~/.kube/client-key.pem")
7    cluster_ca_certificate = file("~/.kube/ca.pem")
8}


1// kubernetes.tf
2// 只能有一個 provider，否則會有 Error: Duplicate provider configuration 的錯誤

重新佈署後錯誤碼就莫名其妙變了:

1[root@master1 terraform]# terraform apply
2helm_release.metrics-server[0]: Refreshing state... [id=metrics-server]
3
4Error: Kubernetes cluster unreachable: invalid configuration: no configuration has been provided, try setting KUBERNETES_MASTER environment variable

照指示加上 環境變數

1export KUBE_CONFIG_PATH=/root/.kube/config

就莫名其妙成功了 = =

補充:

  • k8s provider 說明 https://registry.terraform.io/providers/hashicorp/kubernetes/latest/docs#credentials-config
  • k8s cluster ca https://blog.csdn.net/ll837448792/article/details/103658502

配置 TLS 連接

  1. 查看 kubectl 配置文件，裡面記錄了三個證書和 API server 的地址：

11    user: kubernetes-admin
12  name: kubernetes-admin@kubernetes
13current-context: kubernetes-admin@kubernetes
14kind: Config
15preferences: {}
16users:
17- name: kubernetes-admin
18  user:
19    client-certificate-data: LS0tLS1CR......
20    client-key-data: LS0tLS1CRUdJTiBSU0......
  1. 匯出金鑰及證書
 1[root@testm .kube]# export clientcert=$(grep client-cert ~/.kube/config | cut -d" " -f 6)
 2[root@testm .kube]# echo $clientcert
 3LS0tLS1CRUdJTiBDRVJUSUZJQ0FURS0tLS0=......
 4[root@testm .kube]# export clientkey=$(grep client-key-data ~/.kube/config | cut -d" " -f 6)
 5[root@testm .kube]# echo $clientkey
 6LS0tLS1CRUdJTiBSU0EgUFJJVkFURSB......
 7[root@testm .kube]# export ca=$(grep certificate-authority-data ~/.kube/config | cut -d" " -f 6)
 8[root@testm .kube]# echo $ca
 9LS0tLS1CRUdJTiBDRVJUSUZJQ0FU......
10[root@testm .kube]# echo $clientcert | base64 -d > ./client.pem
11[root@testm .kube]# echo $clientkey | base64 -d > ./client-key.pem
12[root@testm .kube]# echo $ca | base64 -d > ./ca.pem

1[root@testm ~]# kubectl config view | grep server
2    server: https://10.1.5.145:8443
