[Golang] Array v.s. Slice

Publish on: 2019-12-30 Classify at: Programming/Golang
Words: 2364 Read:≈ 5mins

Array

陣列的長度固定，且為陣列類型的一部份。 （[4]int 以及 [5]int 為兩種獨立且不相容的不同type）

宣告及初始化array 

 1
 2// var [len]Type，無初始化值，則初始值為zero value
 3var arr1 [2]int
 4
 5// var [len]Type，並給初始值
 6var arr2 = [2]int{1, 2}
 7
 8// var [...]Type，讓complier自行計算長度
 9var arr3 = [...]int{1, 2}
10
11// := [len]Type，在func中使用簡短宣告符號，無給初始值
12arr4 := [2]int{}
13
14// := [len]Type，並給初始值
15arr5 := [2]int{}
16
17// := [...]Type，讓complier自行計算長度
18arr6 := [...]int{1, 2}

當賦值和傳遞array時，是複製整個陣列內容

 1package main
 2
 3import (
 4	"fmt"
 5)
 6
 7func main() {
 8	arr1 := [2]string{"apple", "banana"}
 9	arr2 := [2]string{}
10
11	arr2 = arr1
12
13	fmt.Printf("arr1: %p, %v \n", &arr1, arr1)
14	fmt.Printf("arr2: %p, %v \n", &arr2, arr2)
15
16	arr3(arr1)
17}
18
19func arr3(x [2]string) {
20	fmt.Printf("arr3: %p, %v \n", &x, x)
21}

執行結果

1arr1: 0x40a0e0, [apple banana] 
2arr2: 0x40a0f0, [apple banana] 
3arr3: 0x40a120, [apple banana]

Array 缺點

上面三個array的位址皆不相同，驗證傳遞時都是複製整個陣列
如此，可發現使用array的兩個弊處

1. array是固定長度, 使用起來很不彈性

2. 傳遞時複製整個組數，array一大時，便會消耗大量記憶體

Warning

所以在大多數的Go code中，slice較常被使用

Slice

slice為Go中的一種資料結構，定義在 src/runtime/slice.go 中： src/runtime/slice.go

1type slice struct {
2	array unsafe.Pointer
3	len   int
4	cap   int
5}

各field代表如下:

1. pointer：指向底層陣列開始的位置

2. len：lenth, 為slice的實際長度

3. cap：capacity, 為當前分給slice的容量大小

宣告slice

slice宣告方式有兩種，一種是直接像array一樣宣告, 只差不須指定大小：

1a := []int{1, 2, 3, 4, 5}
2fmt.Println(len(a))  //5
3fmt.Println(cap(b))  //5

另一種是使用內建函數make
func make([]T, len, cap) []T
make可以事先指定底層陣列長度以及欲使用的容量大小

4b := make([]int, 5, 5) //b == [0, 0, 0, 0, 0]
5fmt.Println(len(b))  //5
6fmt.Println(cap(b))  //5

使用make時，capacity參數可以被省略，而此時的cap==len

7c := make([]int, 5)  //c == [0, 0, 0, 0, 0]
8fmt.Println(len(c))  //5
9fmt.Println(cap(c))  //5

由此得知，slice 本身是一個引用型別，底層會有個指標指向一個array。上面宣告的三個slice a , b , c 都可以用下圖表示

Slice的各種使用

re-slicing 重新切片

使用冒號間隔上下兩參數[:] 可以擷取slice特定範圍的值，
[low-index:high-index]
表示取出從low-index開始的值到到high-index減1的值，high-index的索引代表及於但不包含(up to but not include) 兩個索引值都可以被忽略，忽略時，上邊界默認值為0, 下邊界默認值為len(x)

1x := []int{1, 2, 3, 4, 5}
2y := x[:]  // y == [1 2 3 4 5]
3y = x[:2]  // y == [1 2]
4y = x[2:]  // y == [3 4 5]
5y[2] = 0   // y == [3 4 0] ; x == [1 2 3 4 0]

須特別留意的是

Warning

使用[:]重新切片後，底層的陣列不變，意即y.ptr == x.ptr + 8 (位移了兩位)。 當修改了y[2]的值變是修改了x[4]

若是不想要兩個slice共用同一個array，則要使用下一個內建函數copy

copy 拷貝

使用copy可以複製slice中的array到另一個新的slice的新的array copy(dist-slice, src-slice)

若len(src)小於len(dist)，則會覆蓋目的端的前len(src)個數值：

1x := []int{1, 2, 3, 4, 5}
2y := []int{6, 7, 8}
3copy(x, y) // x == [6 7 8 4 5] 把y複製進x
4y[2] = 0  // y == [6 7 0] ; x == [6 7 8 4 5] 改了y的值，x不受影響，因為底層array不同

若len(src)大於len(dist)，則會複製來源端與len(dist)等長的值：

1x := []int{1, 2, 3, 4, 5}
2y := []int{6, 7, 8}
3copy(y, x) // y == [1 2 3] 把x複製進y

另外也可以結合:指定要複製的位置：

1x := []int{1, 2, 3, 4, 5}
2y := []int{6, 7, 8}
3copy(y, x[2:]) // y == [3 4 5]，把x從index 2開始的值複製進y

Append 擴增

append用來在slice中添加資料，以下為用法：

1
2x := []int{1, 2, 3, 4, 5}
3
4x = append(x, 6)  //append一個數給自己
5
6y := append(x, 7, 8)  //用別的slice append多個數再給自己
7 
8c = append(b, a…) //拿另一個slice append另一個slice再給自己

Warning

問題來了，使用append函數時，底層的array會是一樣的嗎？ 答案是不一定

append底層array的共用與分裂 

1x := []int{}  //len(x) == cap(x) == 0
2y := append(x, 1) //len(y) == 1, cap(y) == 2
3x = append(x, 2)

請問此時的y[0]是多少? y[0] == 1 這個例子下 x & y 分別會指向不同的array，因為在第二行，len(y) != cap(x) 的當下，y底層的array就已經另發配新的位址給新的array了

如果x換成以下宣告:

1x := make([]int, 0, 3)  //len(x) == 0, cap(x) == 3
2y := append(x, 1) //len(y) == len(x) == 1 < cap(x) == 3
3x = append(x, 2) //len(y) == len(x) == 2 < cap(x) == 3

請問此時的y[0]是多少？ y[0] == 2

y[0]會改變的原因是在一開始宣告時記憶體就已經分配cap(x) == 3的空間給底層array，此時使用append加入一個數到array，因為len(y) == len(x) == 1 < cap(x) =3，空間足夠，所以 x及y的指標會指向同一個位址！ 而後再append一個數到array，也會因為空間足夠，所以兩個slices仍在同一個位址。

再衍生另一個問題－ 若原先的最後一行 x = append(x, 2) 改成 x = append(x, 2, 3, 4, 5)，y[0]是多少？

1x := make([]int, 0, 3)
2y := append(x, 1)
3x = append(x, 2, 3, 4, 5)

此時的 y[0] == 1

因為x插入了4個數值，len(x) = 4 > cap(x) = 3，x使用的array便會換一個新的位址，並擴大capacity成len(x)的兩倍，意即cap(x) = 8 而原先的y保持原來的位址以及數值，len(y) = 1 < cap(y) = 3

由此可知，當在使用append時，如果len >= len時，會開另一塊更大記憶體的array，再把原先的array複製進去。
擴大原則：當容量(cap)小於1024時，會擴大兩倍的容量；當大於1024後，擴大 1.25倍。

結論

Note

  1. 使用slice比起使用固定長度的array來得有彈性
  2. slice本身為一個引用型別，底層有指標指向一個array
  3. 重新切片指改變指標位址，底層陣列不變
  4. copy為另外建一個新的底層array
  5. 使用slice時建議事先使用make並規劃好容量cap，這樣在使用append時可以避免反覆重新分配記憶體
