Ingress does not contain a valid IngressClass

Publish on: 2022-02-11 Classify at: Kubernetes
Words: 867 Read:≈ 2mins

問題描述

原 Ingress 配置

以 grafana-ing.tf 為範例 (使用 terraform 自動部署工具安裝)

 1resource "kubernetes_ingress" "grafana_ingress" {
 2  count = var.package.prometheus == true ? 1 : 0
 3
 4  metadata {
 5    name =      "grafana-ingress"
 6    namespace = "monitoring"
 7  }
 8
 9  spec {
10    rule {
11      host = "${var.prometheus-options.host_grafana}.${var.domain}"
12      http {
13        path {
14          backend {
15            service_name = "prometheus-operator-grafana"
16            service_port = 80
17          }
18          path = "/"
19        }
20      }
21    }
22  }
23
24  depends_on = [
25    helm_release.ingress-nginx,
26  ]
27}

發生原因

原因是在 ingress v1.0.0 版之後，ingress 需要加上 ingress class，請參考 github 的 #7341 pull request，如果沒有，controller 會丟 Ignoring ingress because of error while validating ingress class" ingress="k8sNamespace/ingressResourceName" error="ingress does not contain a valid IngressClass" 的錯誤。

Info

An Ingress Class is basically a category which specify who needs to serve and manage the Ingress, this is necessary since in a cluster you can have more than one Ingress controller, each one with its rules and configurations.

解決方法

在 ingress resource 中的 metadata 欄位加上 annotations: kubernetes.io/ingress.class: "nginx" 即可。

 1resource "kubernetes_ingress" "grafana_ingress" {
 2  count = var.package.prometheus == true ? 1 : 0
 3
 4  metadata {
 5    name =      "grafana-ingress"
 6    namespace = "monitoring"
 7    annotations = {
 8      "kubernetes.io/ingress.class" = "nginx"
 9    }
10  }
11
12  spec {
13    rule {
14      host = "${var.prometheus-options.host_grafana}.${var.domain}"
15      http {
16        path {
17          backend {
18            service_name = "prometheus-operator-grafana"
19            service_port = 80
20          }
21          path = "/"
22        }
23      }
24    }
25  }
26
27  depends_on = [
28    helm_release.ingress-nginx,
29  ]
30}

注意

上述方法僅適用於 kubernetes 1.22 版以前。k8s 1.22 版以後請使用 ingressClassName 於 spec 區塊下。請參考 官網 說明。
