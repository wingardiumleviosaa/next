解決 scheduler and controller-manager unhealthy state

Publish on: 2021-08-31 Classify at: Kubernetes
Words: 227 Read:≈ 1mins

Problem

在嘗試更新 Kubernetes 時，下了下面的 command 取得目前集群的組件狀態：

1$ kubectl get cs

發現 controller-mamager 和 scheduler 有 unhealthy 的狀態：

1NAME                 STATUS      MESSAGE                                                                                     ERROR
2controller-manager   Unhealthy   Get http://127.0.0.1:10252/healthz: dial tcp 127.0.0.1:10252: connect: connection refused
3scheduler            Unhealthy   Get http://127.0.0.1:10251/healthz: dial tcp 127.0.0.1:10251: connect: connection refused
4etcd-0               Healthy     {"health":"true"

Reason

這兩個 pod 的非安全端口沒有開啟，健康檢查時報錯，但是由於本身服務是正常的，只是健康檢查的端口沒啟，所以不影響正常使用。

Solution

在所有 Master nodes 上修改下面檔案:

1$ vim /etc/kubernetes/manifests/kube-scheduler.yaml
2$ vim /etc/kubernetes/manifests/kube-controller-manager.yaml

刪掉或註釋掉 - --port=0 (spec->containers->command) 這行

1$ sudo vi /etc/kubernetes/manifests/kube-controller-manager.yaml

重啟 kubelet 服務

1$ sudo systemctl restart kubelet.service

這時10251，10252端口就開啟了，健康檢查狀態也正常了。

1[root@master1 ~]# netstat -tulpn | grep '10251\|10252'
2tcp6       0      0 :::10251                :::*                    LISTEN      11863/kube-schedule
3tcp6       0      0 :::10252                :::*                    LISTEN      11902/kube-controll
