[Linux] LVM- 擴大根目錄 & 移除 pv 碟

Publish on: 2020-12-07 Classify at: OS
Words: 1125 Read:≈ 3mins

前言

結果上次因為 LVM 分的根目錄空間不夠，仗著硬碟多就直接不管空間過剩的 /home，直接找一顆 sdc 掛在相同的根目錄 lv 中。後來系統打算拿一些硬碟做 RAID5，才發現我的系統碟(根目錄)用的是 sda 跟 sdc，跳過了 sdb，強迫症驅使下決定把 sdc 的 pv 移除!!

xfsdump

XFS 是 CentOS 7 預裝的 OS 的檔案系統，在 LVM 中 XFS 只能擴大不能縮小，所以需要利用 xfsdump 備份資料。

目前掛載情況如下，預計把根目錄減掉 5.5T；

安裝xfsdump工具 

1$ yum install xfsdump -y

備份根目錄

將根目錄備份到 /data 下。注意，/data 需與根目錄在不同硬碟，因為根目錄需重建。

進入 Rescue Mode

使用安裝光碟，重開機後進入 Rescue Mode。 進入後按 3 進入 Shell。

縮容

輸入 lvm 進入 lvm 界面。

1sh-4.2# lvm    # 進入 lvm
2lvm> pvs       # 依次掃描 pv, vg, lv
3lvm> vgs
4lvm> lvs
5lvm> lvchange -ay /dev/mapper/centos-root # 啟用根目錄所在的 lv，-a 是啟用 y 是 yes，需啟用 (allocatable=YES) 才能在 shell 中使用該 lv。
6lvm> quit      # 返回 shell

縮小根目錄的 lv。

1sh-4.2# lvreduce -L 4T /dev/mapper/centos-root
2sh-4.2# pvs    # 可以發現已經有空間釋出
3sh-4.2# vgs
4sh-4.2# lvs

開始移除 pv

1sh-4.2# pvmove /dev/sdc1           # 需要看到 no data to move for xxx，非常重要，否則會造成數據遺失
2sh-4.2# pvchange -xn /dev/sdc1     # 註銷
3sh-4.2# pvdisplay                  # 可以發現 /dev/sdc1 的 allocatable 的值為 NO
4sh-4.2# vgreduce centos /dev/sdc1  # 將 pv 從 vg 中刪除
5sh-4.2# pvremove /dev/sdc1         # 刪除 pv

重新格式化根目錄

將 /dev/mapper/centos-root 進行 xfs 格式化，並重新掛載

1sh-4.2# mkfs.xfs /dev/mapper/centos-root
2sh-4.2# mount /dev/mapper/centos-root /
3sh-4.2# xfsrestore /data/root.dump /

離開 shell 重新開機即可。

不負責聲明；；；；

中間不小心迷失方向失敗了，又不好恢復做之前的原狀，就索性重灌了。把前三個硬碟 (sda,sdb,sdc) 都拉進系統碟，重灌結束後發現根目錄一樣分配的很少，其他空間都放到 /home 下面了。所以要乖乖地:

  • 把 /home 縮小
  • 刪除不要用的 pv
  • 最後挪剩下的空間給根目錄
  • 重建兩個目錄的檔案系統

備份 /home 

1$ tar -zcvf /tmp/home.tar.gz /home

卸載 /home 

1$ umount /home

減少 /dev/mapper/centos-home 的 lv 大小 

1$ lvreduce -L 100G /dev/mapper/centos-home

移除 pv 

1$ pvmove /dev/sdc1           # 需要看到 no data to move for xxx，非常重要，否則會造成數據遺失
2$ pvchange -xn /dev/sdc1     # 註銷
3$ pvdisplay                  # 可以發現 /dev/sdc1 的 allocatable 的值為 NO
4$ vgreduce centos /dev/sdc1  # 將 pv 從 vg 中刪除
5$ pvremove /dev/sdc1         # 刪除 pv

重建 file system

擴展根目錄並擴大文件系統

1$ lvextend -l +100%FREE /dev/mapper/centos-root
2$ xfs_growfs /dev/mapper/centos-root

重建 /home 目錄並掛載再還原

1$ mkfs.xfs -f /dev/mapper/centos-home
2$ mount /dev/mapper/centos-home /home/
3$ tar -zxvf /tmp/home.tar.gz -C /home/
4$ rm -rf /tmp/home.tar.gz
